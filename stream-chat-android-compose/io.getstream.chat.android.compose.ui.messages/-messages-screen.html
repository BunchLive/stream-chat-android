Default root Messages screen component, that provides the necessary ViewModels and connects all the data handling operations, as well as some basic actions, like back pressed handling.

Because this screen can be shown only if there is an active/selected Channel, the user must provide a channelId in order to load up all the data. Otherwise, we can't show the UI.

Parameters

channelId

The ID of the opened/active Channel.

messageLimit

The limit of messages per query.

showHeader

If we're showing the header or not.

enforceUniqueReactions

If we need to enforce unique reactions or not.

showDateSeparators

If we should show date separators or not.

showSystemMessages

If we should show system messages or not.

deletedMessageVisibility

The behavior of deleted messages in the list and if they're visible or not.

messageFooterVisibility

The behavior of message footers in the list and their visibility.

onBackPressed

Handler for when the user taps on the Back button and/or the system back button.

onHeaderActionClick

Handler for when the user taps on the header action.

messageId

The ID of the message which we wish to focus on, if such exists.

navigateToThreadViaNotification

If true, when a thread message arrives in a push notification, clicking it will automatically open the thread in which the message is located. If false, the SDK will always navigate to the channel containing the thread but will not navigate to the thread itself.

skipPushNotification

If new messages should skip triggering a push notification when sent. False by default.

skipEnrichUrl

If new messages being sent, or existing ones being updated should skip enriching the URL. If URL is not enriched, it will not be displayed as a link attachment. False by default.
